职引未来
大学生就业信息智能分析平台

整合校内毕业生数据、社会招聘数据与国家宏观就业数据，构建面向高校的就业全链路智能分析系统，为学生、辅导员、就业办三类用户提供差异化服务。

目标用户

为三类核心用户提供差异化服务

在校学生

了解自己的就业竞争力，获取个性化职业发展建议和精准岗位推荐

辅导员 / 就业办

掌握全院学生就业动态，提前识别就业困难群体，生成定向辅导建议

学校管理层

纵向对比历届就业率，为专业调整、课程改革提供数据依据

核心功能模块

覆盖就业全链路的智能化功能

学生信息管理与统计

基础层

结构化录入学生档案，支持 Excel/CSV 批量导入，多维度筛选检索，实时统计就业状态分布。

数据可视化分析

展示层

就业去向分布图、生源地热力地图、行业流向图、薪资分布直方图，支持一键导出 PNG/PDF/Excel。

社会岗位需求分析

供给侧

技能词云、岗位地域热力图、专业-岗位匹配矩阵、学历要求分析，掌握市场需求动态。

AI 就业画像

核心亮点

基于 DeepSeek 大语言模型，生成就业竞争力评分(0-100)，雷达图展示专业匹配度、技能契合度等维度。

智能岗位匹配推荐

RAG 检索

结合向量数据库和 RAG 技术，根据学生档案推荐 Top-N 匹配岗位，给出匹配度说明和差距分析。

就业困难预警

主动干预

AI 自动生成预警标签（技能不足/期望过高/地区限制），批量生成定向辅导建议，推送给辅导员。

AI 简历优化

ATS 模拟

对比目标岗位 JD 关键词，AI 指出简历与岗位的语义差距，给出具体措辞优化建议，提升通过率。

稀缺人才数据分析

数据洞察

展示各省稀缺岗位地图、行业紧缺度排名，为院校专业设置和学生就业方向提供参考。

AI 技术实现路径

大语言模型在就业服务场景的完整调用链路

1

用户数据采集层

学生端采集简历信息、技能标签、职业兴趣；企业端采集岗位要求、薪资范围；院校端采集专业设置、毕业生数据

2

RAG 向量检索增强

将简历文本、企业 JD、稀缺人才数据通过文本嵌入模型向量化，存储至 ChromaDB 向量数据库，支持语义相似度检索

3

System Prompt 注入

构建结构化系统提示词，注入就业评估规则（专业技能权重、设计作品集评估维度）、JSON 输出格式约束

4

LLM 推理层

调用 DeepSeek API (OpenAI Compatible Mode)，temperature=0.2 保证输出稳定性，response_format=json_object 强制 JSON 输出

5

降级兜底策略

API 不可用时启用本地规则引擎（基于关键词匹配、技能标签比对），确保系统在无网络环境下仍能完成基础匹配

6

结果渲染与反馈

将 AI 分析结果映射为可视化卡片（匹配度、技能差距、预警级别），收集用户反馈持续优化模型效果
