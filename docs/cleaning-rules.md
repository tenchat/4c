## 一、通用清洗规则

| 规则类型 | 实现方式 | 示例 |
| --- | --- | --- |
| 去除前后空白 | `strip()` | `" 北京 "` → `"北京"` |
| 统一 NULL 表示 | 映射 `\N`, `null`, `NULL`, `""` → `NULL` | `"null"` → `NULL` |
| 特殊字符清洗 | 换行符 `\n` → 空格，引号标准化 | `"职位要求\n必须本科"` → `"职位要求 必须本科"` |

## 二、字段专项清洗

### 2.1 日期字段

| 源格式             | 目标格式     | 转换规则       | 示例                      |
| ------------------ | ------------ | -------------- | ------------------------- |
| `YYYYMMDD`（整数） | `YYYY-MM-DD` | 字符串切片重组 | `20190325` → `2019-03-25` |

**异常值处理**:

- 若日期值 < `19900101`，设为 `NULL`
- 若日期值 > `20301231`，设为 `NULL`

### 2.2 薪资字段

| 问题类型     | 清洗规则                      | 示例                   |
| ------------ | ----------------------------- | ---------------------- |
| 含单位字符串 | 提取数字部分                  | `"8000元/月"` → `8000` |
| 负值         | 设为 `NULL`                   | `-7249` → `NULL`       |
| 超出合理范围 | 上限 100 万/月，超出设 `NULL` | `9999999` → `NULL`     |

### 2.3 城市码字段

- **源格式**: `"551"` 或 `"551,-,-"`（多值）
- **清洗规则**:
  1. 去除 `-` 占位符
  2. 提取主城市码（第一个有效值）
  3. 需关联 `city_mapping` 表进行解码

**示例**:

```
"551,-,-" → "551"
"530,551,-" → "530"
```

### 2.4 行业字段

- **标准化规则**:
  1. 去除首尾空格
  2. 移除 `/` 重复字符
  3. 建立 `industry_mapping` 表映射非标准名称

**示例**:

```
" 房地产/建筑/建材/工程 " → "房地产/建筑/建材/工程"
"互联网/IT/软件" → "互联网/IT/软件"
```

### 2.5 学历字段

| 原始值     | 标准化值 | 说明   |
| ---------- | -------- | ------ |
| `大专`     | `1`      | 专科   |
| `本科`     | `2`      | 学士   |
| `硕士`     | `3`      | 硕士   |
| `博士`     | `4`      | 博士   |
| `\N`, 空值 | `NULL`   | 无效值 |

### 2.6 薪资码字段（`*_salary_id`）

- **源格式**: 如 `100002000`, `0200104000`
- **问题**: 编码规则不明，需确认
- **建议处理**:
  - 记录原始值暂不解析
  - 或联系数据提供方确认解码规则
  - 临时方案：转为 NULL 待人工确认

### 2.7 技能标签字段（experience）

- **分隔符**: `|`（pipe）
- **清洗规则**:
  1. 按 `|` 分割字符串
  2. 去除空标签
  3. 合并重复项
  4. 存储到 `user_skills` 表（多对多关系）

**示例**:

```
"停车|现场|凤凰||停车" → ["停车", "现场", "凤凰"]
```

## 三、异常值处理策略

| 字段        | 异常特征               | 处理方式               | 原因                     |
| ----------- | ---------------------- | ---------------------- | ------------------------ |
| `min_years` | 值 > 50（如 510, 305） | 设为 `NULL`            | 超出合理工作年限范围     |
| `salary`    | 负值                   | 设为 `NULL`            | 薪资不可能为负           |
| `salary`    | 超出 100 万/月         | 设为 `NULL`            | 超出市场合理范围         |
| `degree`    | 非标准枚举值           | 设为 `NULL` 或建立映射 | 数据质量问题             |
| `birthday`  | > 100 或 < 15（年龄）  | 设为 `NULL`            | 实际字段名为年龄，非生日 |

## 四、编码问题处理

### 4.1 大学生就业选择.csv 系列

**问题**: 文件编码混用 GB2312/GBK/UTF-8，导致中文显示为乱码

**处理方案**:

1. 尝试多种编码格式读取
2. 检测乱码模式并标记
3. 建议联系数据提供方获取正确编码版本

**暂定处理**:

```python
encodings_to_try = ['utf-8', 'gbk', 'gb2312', 'latin1']
for encoding in encodings_to_try:
    try:
        df = pd.read_csv(file, encoding=encoding)
        # 检查是否包含有效中文字符
        if contains_valid_chinese(df):
            break
    except:
        continue
```

## 五、数据质量标记

| 质量问题       | 标记方式                                   | 影响范围     |
| -------------- | ------------------------------------------ | ------------ |
| 编码异常       | 记录原值，标记 `encoding_issue=TRUE`       | 整行或整字段 |
| 薪资码无法解析 | 保留原值，标记 `salary_id_unresolved=TRUE` | 相关字段     |
| 字段值异常     | 设为 `NULL`，记录 `original_value`         | 单字段       |
| 唯一键冲突     | 保留最新值，覆盖旧值                       | 整行         |
